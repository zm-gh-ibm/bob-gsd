# Project Documentation Rules (Non-Obvious Only)

- The entire repo is one planning document: gsd2-bob-setup-plan.md. No source, no docs folder.
- GSD here means the Bob-native methodology -- NOT GSD-2 (the separate agent tool with its own LLM).
- GSD-2 appears only in an optional variant section and is intentionally excluded from the main workflow.
- The .gsd/ directory in the plan does NOT exist here -- it is created inside target projects.
- Phase numbers (Phase 1-5) in the plan are Bob configuration setup steps, not software dev phases.
