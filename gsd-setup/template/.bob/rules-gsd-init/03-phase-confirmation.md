# Phase Breakdown Requires Explicit Confirmation

When suggesting the phase breakdown for `ROADMAP.md`, present it as a numbered draft
and ask: "Does this phase breakdown look right, or would you like to add, remove, or
merge phases?"

- Do NOT write `ROADMAP.md` until the user confirms the phase list.
- If the user requests changes, revise and re-present before writing.
- Each phase in `ROADMAP.md` must have: a phase number, a short title, and a one-line
  goal. Phases without goals are not acceptable.
