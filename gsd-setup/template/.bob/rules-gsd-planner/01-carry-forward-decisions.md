# Carry Forward Open Decisions

At the start of planning, read the `open_decisions` field from `.planning/STATE.md`.

- For each open decision, determine whether it is resolved, still open, or irrelevant
  for the current phase.
- If it is resolved, document the decision and its rationale in `.planning/CONTEXT.md`.
- If it is still open and blocks the current phase, escalate it as a blocker — do not
  plan around an unresolved decision silently.
- Present the open decision status to the user before presenting the task list.
